Core Idea

Semantic Gap은 “검증한 의미”와 “실행된 의미”의 차이다.

프록시, 브라우저, CDN, 프레임워크, 보안 장비는 모두 입력을 해석한다. 문제는 각 계층의 규칙이 완전히 같지 않다는 점이다. 공격자는 이 틈을 이용해 앞단에서는 무해한 요청처럼 보이게 만들고, 뒤단에서는 다른 구조나 권한으로 실행되게 만든다.

Analysis Flow

불일치를 찾는 관찰 순서

  1. Boundary 입력이 어디서 끊기고 이어지는지 확인한다.
  2. Normalization 인코딩, 대소문자, 경로, 유니코드 정규화 순서를 비교한다.
  3. Authority 헤더, 쿠키, 토큰, 캐시 키가 어느 계층에서 신뢰되는지 추적한다.
  4. Policy WAF, 브라우저 정책, 백엔드 권한 검사가 같은 결론을 내는지 본다.
